# Roadmap: Escardar Mail

## Phases
- [ ] **Phase 1: Foundation & Base Identity** - Establish core branding and system environment.
- [ ] **Phase 2: Deep Branding & Access** - Implement proprietary entry points and login experience.
- [ ] **Phase 3: Mail-First Experience** - Pivot UX from cloud storage to professional mail service.
- [ ] **Phase 4: Professional Polish & Onboarding** - Eliminate brand leakage and professionalize user onboarding.

## Phase Details

### Phase 1: Foundation & Base Identity
**Goal**: Establish a working Nextcloud instance with the core Escardar Mail identity and professional communication channel.
**Depends on**: Nothing
**Requirements**: BRAND-01, BRAND-02, BRAND-03, BRAND-04, BRAND-06
**Success Criteria** (what must be TRUE):
  1. User sees "Escardar Mail" and the custom logo in the header.
  2. Browser tab shows the custom Escardar favicon.
  3. UI implements the brand color palette (Cream, Dark Grey/Blue, Light Blue/Grey).
  4. System emails are delivered from the `@escardarmail.com` domain.

### Phase 3: Mail-First Experience
**Goal**: Redirect the user's primary interaction point to the Mail application and align its visuals with the brand.
**Depends on**: Phase 2
**Requirements**: EXP-01, EXP-02, EXP-04
**Success Criteria** (what must be TRUE):
  1. User is automatically directed to the Mail app as the primary landing page upon login.
  2. The Mail UI visual style (colors, spacing) is aligned with the Escardar brand identity.
  3. Only mail-relevant apps are visible in the application menu, hiding generic cloud utilities.

### Phase 4: Professional Polish & Onboarding
**Goal**: Remove all remaining Nextcloud brand leakage and implement a high-touch welcome experience.
**Depends on**: Phase 3
**Requirements**: BRAND-07, EXP-05
**Success Criteria** (what must be TRUE):
  1. Automated system emails contain no mentions of "Nextcloud".
  2. New users are guided through a branded onboarding sequence upon their first login.
